এইবার প্রাক্টিক্যালি আমরা দেখব কিভাবে JavaScript/TypeScript এর OOP এর কাজ করা হয়
, প্রথমত শুরু করব ক্লাস এর সিনটেক্স দিয়ে ।

### Class

#### Basic Class 

```js
class Person {}
```


#### JavaScript দিয়ে বানানো 
```js
class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }

greet() { return "Hello, my name is " + this.name + " and I am " + this.age + "
years old."; } }

const person1 = new Person("John", 25); console.log(person1.greet()); 
```

#### TypeScript দিয়ে বানানো 
```ts 
class PersonTS {
  name: string;
  age: number;

constructor(name: string, age: number) { this.name = name; this.age = age; }

greet(): string { return "Hello, my name is " + this.name + " and I am " +
this.age + " years old." } }

const person2 = new PersonTS("Alice", 30); console.log(person2.greet());
```


### বিস্তারিত প্রতিটি ক্লাসের ভাগ করি এখন 
#### ক্লাস এর জন্য কয়েকটা জিনিস প্রয়োজন । 

 1. Class  : হল কোনো  অবজেক্ট বানানোর জন্য ফ্যান্টরি  নীল-নকশা 
 2. constructor : এটি Class এর কোন প্যারামিটার দিলে সেটা গ্রহন ও সেট করে ক্লাসের প্রোপার্টি  হিসেবে । Class ডিফাইন করলে সর্বপ্রথম এটা রান হয় । 
 3. methods :  Function কে ক্লাস এর ক্ষেত্রে মেধড বলা হয় । এক্ষেত্রে এই Function মূলত ক্লাস এর ভিতরে লেখা ও রান হয় ।  যেমন উপরের greet() এটা একটি মেথদ তবে Class ছাড়া বানানো হলে এটাকে Function বলা হত কিন্তু যেহেতু এটা একটা ক্লাসের ভিতর তাই এটাকে Method বলা হবে । 

 4. properties : হল কোনো অবজেক্টের ডেটা বা state  সংরক্ষণকারী ভেরিয়েবল। যেমনঃ name, age এসব ভ্যারিয়েবলগুলো, সহজ ভাষায় ক্লাসের ভিতরের   ভ্যারিয়েবলগোলোকে properties বলা হয় । 
 5. Access modifiers  : public, private, ও protected এসব tag দিয়ে কোন  properties কে কেমন  শেয়ার বা রিস্ট্রেশন রেখে দেয়ার জন্য ব্যবহার করা হয় । এসবের মাধ্যমেরি Encapsulation নিশ্চিত করা হয় ।
