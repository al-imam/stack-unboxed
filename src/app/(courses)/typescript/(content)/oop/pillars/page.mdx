## 3. inheritance :
 অন্য কোন ক্লাস এর মেথদ বা প্রোপার্টি কোন ক্লাসে নিয়ে আসে তা একসেস (Access)  বা ব্যবহার করাকে inheritance বলা হয় । উদাহরণ সহ এখন দেখিঃ

<CodeBlock language="TypeScript" 
code={`
 class Animal {
  name: string;

 constructor(name: string) {
  this.name = name;
}

  makeSound(): void {
  console.log("Animal makes a sound");
   }
 }

 // class that inheriting Animal
 class Dog extends Animal {
makeSound(): void {
console.log("Woof! Woof!");
   }
}`} />

   এখানে Dog ক্লাস Animal ক্লাসকে `extends` keyword এর মাধ্যমে এর property এবং methods extends করছে । এক্ষাত্রে Animal হল Parent Class ও Dog হল Child Class ।  
   এটাকে এটাকে কিভাবে ব্যবহার করব তা দেখি । 

<CodeBlock language="TypeScript" 
  code={`
 // Create an instance of Dog
 const myDog = new Dog("Buddy");
 console.log(myDog.name); // Output: Buddy
 myDog.makeSound(); // Output: Woof! Woof!
`}
   /> 

যদিও Dog Class এর নিজের কোন `makeSound` নামের কোন method নাই কিন্তু তবুও আমরা ব্যবহার করতে পারছি কারন এটা তার প্যারেন্ট অর্থাৎ Animal Class এর থেকে নিয়েছে । 




## 4. polymorphism :

polymorphism মূলত Greek শব্দ থেকে এসেছে । এক্ষেত্রে শব্দ খন্ডালে পাইঃ 

 - Poly : many (অনেক)
 - Morph : forms(আকার/গঠন)

যার শব্দের মানে হল `Many Forms` বা  `অনেক আকার/গঠন`  । 

### polymorphism এর প্রয়োজনীয়তাঃ 
 -  কোড রি-ইউজিবিলিটিঃ একটা সিঙ্গেল মেথড বা ফাংশন যা একাধিক ক্ষেত্রে একাধিক ভাবে ব্যবহার করা যায় । 
 - এক্সটেনসিবিলিটিঃ নতুন ফিচার সহজেই এড করা যায়, এক্সিস্টিং কোড  পরিবর্তন না করেই । 
 - মেইন্টেনেবিলিটিঃ কোড সহজেই মেইন্টেইন করা যায়, এক্সিটিং কোড যেহেতু পরিবর্তন করার প্রয়োজন হয় না ও সহজেই কোড রিড হয় যায় ।
